# Chalk 제품 스펙 문서

## 1. 제품 개요

### 한 줄 정의
> **Chalk = 과외 선생님이 수업 끝나고 30초 안에 학부모에게 피드백 보내고, 그 기록이 검증된 포트폴리오가 되는 앱**

### 핵심 가치 제안
| 대상 | Pain Point | Chalk 솔루션 |
|------|------------|--------------|
| 튜터 | "피드백 쓰기 귀찮아" | 30초 완료 (체크 + 음성 → AI 변환) |
| 튜터 | "내 경력 증명할 방법 없어" | 누적 데이터가 포트폴리오 |
| 튜터 | "커리큘럼 짜기 어려워" | AI가 선수학습 맵 기반 자동 생성 |
| 학부모 | "우리 애 수업 어떻게 했는지 모르겠어" | 실시간 이미지 리포트 |

## 2. 핵심 기능

### 2.1 수업 기록 (30초 완료)
```
1. 오늘 수업한 학생 선택
2. 학습 목표별 달성도 체크 (상/중/하)
3. 한 줄 메모 or 음성 메모
4. [완료] → AI가 정중한 문장으로 변환 → 카톡 전송
```

### 2.2 신규 학생 원클릭 세팅
```
입력: 학년, 과목, 현재 수준, 목표
출력:
  - 예상 결손 단원 (선수학습 맵 기반)
  - 주차별 커리큘럼 초안
  - 학부모용 초기 진단 리포트
```

### 2.3 수업 전 AI 브리핑
```
수업 10분 전 알림:
  - 지난 수업 요약
  - 오늘 목표
  - 복습 필요한 포인트
  - 튜터 힌트 ("이 학생은 시각적 설명에 반응 좋음")
```

### 2.4 학부모 메시지 플로우
```
[수업 직후] 텍스트 메시지
  - AI가 튜터 메모를 학부모용 문장으로 변환
  - 핵심만: 오늘 배운 것 + 잘한 점 + 복습 포인트
  - 카톡에서 바로 읽을 수 있는 길이

[수업 3일 후] 이미지 카드
  - 시각적 진도 요약
  - 학부모가 저장/공유하기 좋은 포맷
  - "기록이 되는" 느낌

[매주 일요일] 주간 종합 리포트 이미지
  - 그 주 모든 수업 종합
  - 성취 하이라이트 + 다음 주 목표
  - 튜터 브랜딩 포함

* 외부 링크 없음 - 카톡 안에서 완결
* 학부모가 "클릭"할 필요 없음 (이미지만 보면 됨)
```

### 2.5 튜터 포트폴리오
```
외부 공개 가능한 프로필:
  - 총 수업 횟수 (검증됨)
  - 학생 유지율
  - 연속 기록 일수 (스트릭)
  - 활동 잔디 (GitHub 스타일)
  - "Chalk 인증 튜터" 뱃지
```

## 3. 선수학습 연계 맵 (중등 수학)

### 3.1 데이터 출처
- [2022 개정 교육과정 수학과](https://namu.wiki/w/2022%20개정%20교육과정/수학과)
- [NCIC 국가교육과정 정보센터](https://ncic.re.kr)
- [서울시교육청 연수 자료](https://www.sen.go.kr)

### 3.2 영역별 선수학습 맵

#### 수와 연산
```
초6 분수/소수 계산
    ↓
중1 정수와 유리수
    ↓
중2 유리수와 순환소수
    ↓
중3 제곱근과 실수
```

#### 문자와 식 → 방정식
```
초6 비와 비율
    ↓
중1 문자와 식
    ├→ 중1 일차방정식
    │      ↓
    │  중2 연립방정식
    │      ↓
    └→ 중3 이차방정식 ←── 중3 인수분해
                              ↑
                         중2 다항식 곱셈
                              ↑
                         중1 문자와 식
```

#### 함수
```
초6 비례식
    ↓
중1 좌표평면과 그래프
    ↓
중2 일차함수 ←── [선수] 중1 일차방정식
    ↓
중3 이차함수 ←── [선수] 중3 이차방정식
```

#### 도형
```
초6 도형의 넓이
    ↓
중1 기본 도형 (점, 선, 면, 각)
    ↓
중2 삼각형/사각형 성질
    ↓
중2 도형의 닮음
    ↓
중3 피타고라스 정리
    ↓
중3 삼각비
```

### 3.3 흔한 결손 패턴 (튜터 커뮤니티 기반)

| 현재 막히는 곳 | 실제 결손 위치 | 출처 |
|---------------|---------------|------|
| 일차방정식 | 음수 계산, 이항 개념 | 오르비, 김과외 커뮤니티 |
| 일차함수 | 좌표평면 이해, 변수 개념 | 튜터 인터뷰 |
| 이차방정식 | 인수분해 (약수/배수 결손) | 교육과정 연계표 |
| 이차함수 | 이차방정식, 좌표평면 | 교육과정 연계표 |
| 도형 증명 | 논리적 사고력, 용어 이해 | 튜터 인터뷰 |

### 3.4 AI 진단 로직 예시

```typescript
interface DiagnosticInput {
  grade: string;        // "중2"
  subject: string;      // "수학"
  currentTopic: string; // "일차함수"
  currentLevel: "상" | "중" | "하";
  targetTopic?: string; // "이차함수" (optional)
}

interface DiagnosticOutput {
  estimatedGap: string;           // "중1 수준 (-1년)"
  missingPrerequisites: string[]; // ["좌표평면", "일차방정식"]
  recommendedStartPoint: string;  // "좌표평면 복습"
  estimatedWeeks: number;         // 16
  curriculum: CurriculumWeek[];
  warnings: string[];             // ["일차함수 어려운 학생 78%가 변수 개념 결손"]
}
```

## 4. 학부모 메시지 스펙

### 4.1 수업 직후 텍스트 (Stage 1)

**발송 시점**: 수업 종료 직후 (0-30분 내)
**채널**: 카카오톡 또는 SMS

```
안녕하세요, {학생이름} 어머님.
{튜터이름} 선생님입니다.

📚 오늘 {과목} 수업에서는 {오늘 학습 내용}을(를) 다루었습니다.

{학생이름} 학생이 {잘한 점/피드백}.

{복습 필요한 점 또는 다음 수업 예고}

감사합니다.
{튜터이름}
```

**특징**:
- 짧고 간결 (카톡에서 스크롤 없이 읽을 수 있는 길이)
- 이모지 최소화 (학부모 연령층 고려)
- 정중한 존댓말 (AI가 튜터 메모를 변환)

### 4.2 3일 후 이미지 카드 (Stage 2)

**발송 시점**: 수업 3일 후 (기억 강화 타이밍)
**채널**: 카카오톡 이미지

```
┌─────────────────────────────┐
│                             │
│  📊 {학생이름}의 학습 기록   │
│  ─────────────────────      │
│  {날짜} {과목} 수업          │
│                             │
│  ┌─────────────────────┐   │
│  │ ✅ 일차방정식 풀이   │   │
│  │    ████████████ 95% │   │
│  │                     │   │
│  │ 🔄 이항 정리         │   │
│  │    ████████░░░░ 70% │   │
│  └─────────────────────┘   │
│                             │
│  💡 잘한 것: {잘한 점}       │
│  📝 복습: {복습 포인트}      │
│                             │
│  ─────────────────────      │
│  🌱 Chalk · {튜터이름}       │
│                             │
└─────────────────────────────┘
```

**디자인 스펙**:
- 크기: 1080x1350px (인스타그램 비율)
- 배경: #1A1A1A (다크 모드)
- 강조색: #10B981 (에메랄드)
- 폰트: Pretendard
- 저장/공유하기 좋은 비주얼

**왜 3일 후?**
- 에빙하우스 망각곡선: 3일 후 복습 리마인더
- 학부모가 "이 튜터는 follow-up도 한다" 느낌
- 수업 직후 과도한 메시지 피함

### 4.3 주간 종합 리포트 (Stage 3 - 매주 일요일)

```
┌─────────────────────────────┐
│                             │
│     📊 이번 주 학습 현황     │
│        {학생이름}           │

│   │  🔥 {N}회 수업 완료 │   │

│   │  ✅ {완료 단원1}    │   │

│   │  🔄 {진행중 단원}   │   │
│   └─────────────────────┘   │
│                             │
│   진도 {진도}% █████░░░░   │
```

## 5. 기술 스택

- React Native + Expo

- React Native Reanimated

- Next.js 16

## 6. MVP 범위 (Phase 1)

### 포함
- [ ] 학생 관리 (CRUD)
- [ ] 수업 기록 (학습 목표 체크 + 메모)
- [ ] AI 피드백 변환 (메모 → 정중한 문장)
- [ ] 카카오톡 전송 (텍스트 + 이미지)
- [ ] 튜터 포트폴리오 기본 (수업 횟수, 잔디)
- [ ] 선수학습 기반 진단 (중등 수학)

### 미포함 (Phase 2+)
- 수업 녹음/분석
- 학부모 전용 앱
- 과외 매칭
- 결제 시스템

## 7. 경쟁 우위 (Moat)

### 즉시 차별화
| 기능 | 기존 대안 | Chalk |
|------|----------|-------|
| 피드백 자동화 | ChatGPT 복붙 | 원클릭 + 카톡 연동 |
| 커리큘럼 생성 | 직접 짜기 (5시간) | 30초 자동 생성 |
| 학부모 리포트 | 직접 만들기 | 이미지 카드 자동 |

### 장기 해자
| 해자 유형 | Chalk 적용 |
|-----------|------------|
| 네트워크 효과 | 학부모↔튜터 양면 시장 |
| 전환 비용 | 누적 데이터/평판 잃음 |
| 데이터 자산 | 학습 패턴 집단 지성 |
